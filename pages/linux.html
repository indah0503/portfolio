Quiz · Troubleshooting

Apache2 Service Failure & 403 Fix

Apache2 systemctl vim mod_ssl VirtualHost
01Detected connection failure with curl http://localhost — port 80 unreachable.
02Checked systemctl status apache2: syntax error on line 148 of apache2.conf — bare SSLEngine directive with no value and no SSL module loaded.
03Fixed by setting SSLEngine off and enabling the SSL module via a2enmod ssl. Alternative: comment out the line entirely.
04After restart, 403 persisted — no <Directory> block in the VirtualHost. Apache defaults to deny-all.
05Added Require all granted inside a <Directory /var/www/html> block in 000-default.conf, restarted Apache. Index page served successfully.
Resolved — curl returns HTTP 200

02Copied all files with mode 0600 to backup/ using -exec cp {} dest \;, preserving original permissions.

Challenge 3 · Storage

02Formatted with mkfs.ext4 -L appcache /dev/sdd1; confirmed UUID with blkid.
03Created mount point /app and mounted with mount LABEL=appcache /app.
04Added LABEL=appcache /app ext4 defaults,nofail 0 2 to /etc/fstab; tested with mount -a.
